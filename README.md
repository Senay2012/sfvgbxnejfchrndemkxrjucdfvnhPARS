# 📡 Telegram Parser PRO — v4.0

Бот для парсинга пользователей из Telegram-каналов.  
Два режима: живой мониторинг в реальном времени и пакетный сбор базы из истории сообщений.  
Управление через inline-кнопки в Telegram-боте.

---

## ⚙️ Установка

```bash
pip install -r requirements.txt
python parserV.py
```

---

## 📁 Структура файлов

| Файл | Описание |
|---|---|
| `parserV.py` | Основной скрипт |
| `bot_config.json` | Конфигурация (каналы, фильтры, настройки) |
| `crypto_users.csv` | База собранных пользователей (CSV) |
| `collected_base.txt` | Результат сбора базы (список @username) |
| `session_name.session` | Сессия UserClient (Telethon-аккаунт) |
| `bot_session.session` | Сессия BotClient |

---

## 🔧 Конфигурация (`bot_config.json`)

| Параметр | Тип | Описание |
|---|---|---|
| `monitoring_active` | bool | Включён ли мониторинг в реальном времени |
| `send_notifications` | bool | Отправлять уведомления о новых юзерах |
| `only_with_username` | bool | Сохранять только пользователей с @username |
| `only_premium` | bool | Сохранять только Telegram Premium |
| `target_chat_id` | string | ID чата куда слать уведомления мониторинга |
| `collect_chat_id` | string | ID чата куда слать результат сбора базы |
| `collect_limit` | int | Лимит сообщений на канал при сборе (10–5000) |
| `monitored_channels` | list | Каналы для мониторинга в реальном времени |
| `collect_channels` | list | Каналы для пакетного сбора базы |

Получить `API_ID` и `API_HASH`: [my.telegram.org](https://my.telegram.org)

## 🧩 Режимы работы

### 1. 📡 Мониторинг в реальном времени

Подписывается на новые сообщения в указанных каналах.  
Каждый новый уникальный пользователь:
- Сохраняется в `crypto_users.csv`
- Отправляется уведомление в `target_chat_id`

Мониторинг работает непрерывно пока запущен скрипт.  
Повторные записи исключены — каждый user ID сохраняется один раз.

Управление через бота: `МОНИТОРИНГ → Поставить на паузу / Возобновить`

---

### 2. 🗂 Сбор базы

Парсит историю сообщений из выбранных каналов **параллельно**.  
Результат — файл `collected_base.txt`: один `@username` на строку.

Запуск через бота: `СБОР БАЗЫ → ЗАПУСТИТЬ СБОР БАЗЫ`

По завершении файл отправляется в `collect_chat_id` и администратору.

## 📊 База данных (`crypto_users.csv`)

CSV-файл с колонками:

| Канал | Username | Имя | Премиум | ID | Время |
|---|---|---|---|---|---|
| https://t.me/channel | @user | Имя | Да/Нет | 123456789 | 2024-01-01 12:00:00 |

---

## 💾 Экспорт

Доступен через бота (`Экспорт базы`):

- **TXT** — только `@username`, один на строку (для рассылки)
- **CSV** — полная база со всеми полями

---

## ⚙️ Настройки фильтров

Настраиваются через бота (`Настройки фильтров`):

- **Только с @username** — игнорирует аккаунты без юзернейма
- **Только Premium** — сохраняет только Telegram Premium пользователей
- **Чат уведомлений** — куда слать уведомления мониторинга
- **Чат базы** — куда слать файлы при сборе и экспорте

---

## 📋 Управление каналами

Через бота (`Каналы`):

- Добавить/удалить каналы мониторинга
- Добавить/удалить каналы для сбора базы
- Поддерживаются форматы: `https://t.me/channel`, `@channel`, `channel`

---

## 🚀 Запуск

При первом запуске потребуется авторизация UserClient — введите номер телефона и код из Telegram.  
Сессия сохраняется в `session_name.session` и при следующих запусках авторизация не нужна.

```bash
python parserV.py
```

Затем отправить боту `/menu` или `/start`.
